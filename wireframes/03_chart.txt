NOTE ON CHART ENGINE
─────────────────────────────────────────────────────────────────────────────
Primary: TradingView Charting Library (https://tradingview.com/charting-library)
  - Free for non-commercial / personal use (apply for access at tradingview.com)
  - Access granted to private GitHub repo: tradingview/charting_library
  - Requires a JavaScript DataFeed adapter (IBasicDataFeed) that calls our
    backend /historical and /instruments endpoints to supply OHLCV data
  - ALL indicators (100+), drawing tools (50+), sub-panes are BUILT IN to TV
  - Drawings are persisted to our backend via TV's saveChartToServer /
    loadChartFromServer callbacks, which call our /charts/{token}/drawings API
  - Chart state (zoom, indicators, layout) saved via TV's own save mechanism

Fallback: TradingView Lightweight Charts (open source, MIT licence)
  - No access approval needed; install via npm
  - Simpler UI; indicators computed by backend and overlaid as series
  - Drawing tools implemented manually; fewer built-in options
  - Use this only if Charting Library access is not granted
─────────────────────────────────────────────────────────────────────────────

================================================================================
DATAFEED ADAPTER (frontend only — connects TV to StockPilot backend)
================================================================================

The TradingView Charting Library requires a JavaScript IBasicDataFeed implementation:

resolveSymbol(symbolName)
  → calls GET /instruments/search?q={symbolName}
  → returns SymbolInfo (name, description, type, exchange, minmov, pricescale…)

getBars(symbolInfo, resolution, from, to, onResult)
  → calls GET /historical/{instrument_token}?interval={resolution}&from={from}&to={to}
  → converts backend candle array to TV Bar array: { time, open, high, low, close, volume }

subscribeBars(symbolInfo, resolution, onRealtimeCallback, uid)
  → v1: polls GET /historical every 60 seconds for the last candle (no WebSocket)
  → v2: upgrade to WebSocket stream

unsubscribeBars(uid)
  → clears the polling interval

searchSymbols(userInput, exchange, type, onResult)
  → calls GET /instruments/search?q={userInput}&exchange={exchange}

================================================================================
FALLBACK: LIGHTWEIGHT CHARTS LAYOUT (if Charting Library access not granted)
================================================================================

If TradingView Charting Library access is denied, TradingView Lightweight Charts
(open source, MIT licence) is used. Layout reverts to the custom toolbar approach:

┌──────────────────────────────────────────────────────────────────────────────────┐
│  [≡]  StockPilot      ● MARKET OPEN                              [↺]  ABC123    │
├──────┬───────────────────────────────────────────────────────────────────────────┤
│      │                                                                            │
│ [D]  │  [ 🔍 Search symbol... ▾ ]   NSE ▾   [ 5m  15m  30m  1H  D ]            │
│ [C]  │  ─────────────────────────────────────────────────────────────────────── │
│ [K]  │  Drawing: [↖] [↗] [↔] [⬛] [💬] [🗑 Clear]  [💾 Save drawings]          │
│ [S]  │  ─────────────────────────────────────────────────────────────────────── │
│ [B]  │  [ Lightweight Charts candlestick canvas — no built-in indicator UI ]    │
│ [O]  │  [ Indicators added as series via POST /charts/indicators/compute ]       │
│ [A]  │                                                                            │
│      │  ┌──────────────── INDICATOR PANEL ─────────────────────────────────┐    │
│      │  │ [ + Add Indicator ▾ ]         [ Manage ]                          │    │
│      │  │ Active: EMA(20) · EMA(50) · Bollinger Bands · RSI(14) · MACD     │    │
│      │  └──────────────────────────────────────────────────────────────────┘    │
│      │                                                                            │
└──────┴───────────────────────────────────────────────────────────────────────────┘

In the Lightweight Charts fallback, indicators are fetched from backend:
  GET /charts/indicators/compute?instrument_token=&interval=&indicators=EMA_20,...
Backend computes series using pandas-ta and returns time-series arrays.
Frontend renders them as overlay/sub-pane series on the Lightweight Chart.
