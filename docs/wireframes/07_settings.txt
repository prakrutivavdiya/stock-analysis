KITE SESSION EXPIRED — PROFILE STATE
================================================================================

  When kite_access_token has expired (detected via 403 from Kite API):

  ┌─────────────────────────────────────────┐
  │   ○ Expired   re-authenticate ↗          │   ← Kite session status row
  └─────────────────────────────────────────┘

  Full banner also shown at top of every page (see 00_app_shell.txt):
  ┌──────────────────────────────────────────────────────────────────┐
  │  ⚠  Your Kite session has expired — live data is unavailable.    │
  │     [ Re-authenticate with Kite ]                   [✕ dismiss]  │
  └──────────────────────────────────────────────────────────────────┘


================================================================================
DATA SOURCES FOR PROFILE FIELDS
================================================================================

  Field               Source
  ──────────────────  ──────────────────────────────────────────────────────
  Full name           GET /api/v1/auth/profile → users.kite_user_name
  Kite user ID        GET /api/v1/auth/profile → users.kite_user_id
  Email               Kite GET /user/profile → email (stored on first login)
  Exchanges           Kite GET /user/profile → exchanges list
  Products            Kite GET /user/profile → products list
  Kite session status Backend: kite_token_expires_at vs current time (IST)
  JWT status          Frontend: decode JWT exp claim from httpOnly cookie
                      (backend /auth/me endpoint returns expiry info)

  Profile data is fetched once on Settings page open via GET /api/v1/auth/profile.
  No sensitive tokens are exposed to the frontend.
